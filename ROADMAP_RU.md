## 1. Важное примечание

Проект DuoNet находится на стадии **поиска команды для реализации**. Вся документация (White Paper, Technical Spec, Economic Model) полностью готова, но код ещё не написан.

Указанные ниже сроки отсчитываются **от момента формирования команды**. Пока команды нет, проект существует только как концепция и документация.

---

### Этап 1: Прототипирование (Testnet Alpha)

*Длительность: 2-3 месяца после формирования команды*

**Цель:** Создать работающий прототип для тестирования базовой функциональности.

#### Технические задачи:

| Задача                        | Технологии            | Сложность | Ожидаемый результат                           |
|-------------------------------|-----------------------|-----------|-----------------------------------------------|
| Сервер знакомств (rendezvous) | Python + FastAPI/gRPC | Низкая    | Рабочий сервер для регистрации и поиска пиров |
| Базовая идентификация         | Python + PyNaCl       | Низкая    | Генерация ключей Ed25519, сид-фразы BIP39     |
| Прототип блокчейна            | Python                | Средняя   | Транзакции, блоки, хеширование (без P2P)      |
| TUI-клиент (минимальный)      | Python + Textual      | Средняя   | Отправка сообщений через mock-сервер          |
| Эмуляция NAT-серверов         | Python + Docker       | Средняя   | Тестирование механик для серверов за NAT      |

**Результат этапа:**
- Работающий прототип сервера знакомств
- Возможность создавать ключи и подписывать транзакции
- TUI-клиент для ручного тестирования
- Публикация кода в открытом репозитории

---

### Этап 2: Альфа-версия (Testnet Beta)

*Длительность: 3-4 месяца после Этапа 1*

**Цель:** Реализовать ядро блокчейна и P2P-сеть.

#### Технические задачи:

| Задача                 | Технологии            | Сложность | Ожидаемый результат                     |
|------------------------|-----------------------|-----------|-----------------------------------------|
| Интеграция libp2p      | Go/Rust или py-libp2p | Высокая   | P2P-сеть с DHT и GossipSub              |
| Полноценный консенсус  | Go/Rust               | Высокая   | Proof-of-Trust с расчётом веса серверов |
| Механизм эмиссии       | Go/Rust               | Средняя   | Прогрессивная эмиссия с 50% сжигания    |
| Распределение комиссий | Go/Rust               | Средняя   | 70/10/20 (инициатор/валидатор/сжигание) |
| Улучшенный TUI-клиент  | Python + Textual      | Средняя   | Поддержка кошелька, истории             |
| Базовый GUI-клиент     | PySide6 / Qt          | Средняя   | Первая версия для тестирования          |

**Результат этапа:**
- Работающая P2P-сеть с 10-20 тестовыми серверами
- Консенсус Proof-of-Trust в действии
- Возможность отправлять транзакции и видеть их в блоках
- Закрытое тестирование с первыми участниками

---

### Этап 3: Бета-версия (Mainnet Candidate)

*Длительность: 2-3 месяца после Этапа 2*

**Цель:** Подготовить сеть к запуску в основной сети.

#### Технические задачи:

| Задача                         | Технологии       | Сложность | Ожидаемый результат                |
|--------------------------------|------------------|-----------|------------------------------------|
| Смарт-контракты (escrow)       | VM-движок        | Высокая   | Работающие контракты с невозвратом |
| Двойной ключ (Double Key)      | Python/C++       | Средняя   | Интеграция доп. фразы в шифрование |
| Аудит безопасности             | Внешние эксперты | Высокая   | Исправление уязвимостей            |
| Нагрузочное тестирование       | -                | Средняя   | Оптимизация производительности     |
| Полноценный GUI-клиент         | PySide6 / Qt     | Средняя   | Готовый клиент для пользователей   |
| Документация для пользователей | Markdown         | Низкая    | Инструкции, FAQ                    |

#### Организационные задачи:

| Задача                          | Ответственные | Ожидаемый результат                    |
|---------------------------------|---------------|----------------------------------------|
| Запуск основной сети            | Команда       | Mainnet с реальными монетами           |
| Привлечение первых 100 серверов | Маркетинг     | Стартовая сеть с достаточной мощностью |
| Начальное распределение монет   | Команда       | Прогрессивная эмиссия по шкале         |
| Биржевые листинги (опционально) | Маркетинг     | Торговля DNET на небольших биржах      |

**Результат этапа:**
- Работающая основная сеть (mainnet)
- Первые 100 серверов от приглашённых майнеров
- Начало обращения монет DNET
- Сообщество ранних пользователей

## 6. Как повлиять на приоритеты

Дорожная карта — не догма, а план. Приоритеты могут меняться в зависимости от:

- **Мнения сообщества** — что важнее для пользователей
- **Возможностей команды** — какие технологии доступны
- **Рыночной ситуации** — что актуально сейчас
- **Наличия финансирования** — гранты, инвестиции

Если вы хотите повлиять на приоритеты:
1. Присоединяйтесь к обсуждению (чат / issues)
2. Аргументируйте свою позицию
3. Предложите помощь в реализации
