Благодарим вас за интерес к проекту DuoNet! Это открытый проект, и мы приветствуем любую помощь — от исправления опечаток до разработки ядра блокчейна.

## 1. О проекте

DuoNet — это децентрализованная P2P-сеть общения с экономическим стимулированием участников. Проект находится на стадии поиска команды для реализации. Вся документация (White Paper, Technical Spec, Economic Model) уже подготовлена и доступна в репозитории.

### 2.3 🔧 Разработка

Мы ищем разработчиков для реализации проекта. Если вы хотите участвовать в создании кода:

#### Технологии, которые нам нужны:

| Область | Технологии | Сложность |
|---------|------------|-----------|
| **P2P-сеть** | libp2p (Go/Rust), NAT Traversal, DHT | Высокая |
| **Блокчейн-ядро** | Python (прототип), Go/Rust (prod) | Высокая |
| **Криптография** | Double Ratchet, libsodium, Ed25519 | Высокая |
| **Клиент (TUI)** | Python + Textual | Средняя |
| **Клиент (GUI)** | PySide6 / Qt | Средняя |
| **Сервер знакомств** | Python + FastAPI/gRPC | Низкая |
| **Документация** | Markdown, английский/русский | Низкая |

#### Что можно сделать уже сейчас:

1. **Сервер знакомств (rendezvous)** — отдельный микросервис на Python/FastAPI. Не требует блокчейна, можно начать с него.
2. **TUI-клиент** — базовый интерфейс для отправки сообщений через mock-сервер.
3. **Прототип блокчейна** — транзакции, блоки, хеширование (без P2P).
4. **Инструменты для тестирования** — генерация ключей, подпись транзакций.

## 5. Процесс работы

### Если вы нашли баг или хотите предложить улучшение:

1. Проверьте, нет ли уже такого issue в репозитории
2. Создайте новое issue с подробным описанием
3. Если это баг — укажите шаги для воспроизведения
4. Если это предложение — объясните, почему это улучшит проект

### Если вы хотите внести изменения в код:

1. Форкните репозиторий
2. Создайте ветку для своей работы (`feature/your-feature-name`)
3. Пишите код, соблюдая стиль (PEP 8 для Python)
4. Добавьте тесты, если это возможно
5. Сделайте pull request в основной репозиторий
6. Опишите, что вы изменили и почему

## 6. Стиль кода и соглашения

### Python

- Соблюдайте [PEP 8](https://peps.python.org/pep-0008/)
- Используйте докстринги для классов и функций
- Типизация приветствуется (type hints)

### Прочее

- Комментарии в коде — на английском (для международной команды)
- Документация — на русском и английском
- Имена переменных — осмысленные, без сокращений

Весь код проекта будет распространяться под лицензией **MIT**. Документация — под лицензией **CC BY-NC 4.0**.
