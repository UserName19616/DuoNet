## О проекте

DuoNet — это не просто мессенджер и не просто блокчейн. Это экосистема свободного общения, где каждый участник может найти свою роль:

- **Обычные пользователи** получают бесплатный, безопасный и приватный мессенджер с уникальной защитой "двойной ключ"
- **Энтузиасты** запускают серверы (даже на домашних ПК за NAT), зарабатывают монеты DNET и помогают сети расти
- **Бизнес** использует революционные смарт-контракты с невозвратом средств для безопасных сделок
- **Разработчики** создают новые сервисы на открытой платформе

Название DuoNet отражает двойственность, пронизывающую весь проект: два типа участников (серверы и клиенты), два уровня защиты (шифрование + дополнительная фраза), два источника дохода для серверов (майнинг и инициация транзакций).

## Ключевые инновации

### 🔥 Escrow с невозвратом (уникально!)
Первый в мире механизм смарт-контрактов, где не договорившиеся стороны теряют всё. Средства блокируются на специальном адресе и могут быть получены продавцом только при предъявлении правильной фразы. Если фраза не предъявлена до дедлайна — средства сгорают. Это создаёт идеальный стимул для выполнения обязательств: либо сделка состоялась, либо деньги исчезают навсегда.

### 🔐 Двойной ключ (уникально!)
Стандартное E2EE-шифрование (Double Ratchet) дополняется пользовательской фразой, которую собеседники передают друг другу лично или по телефону. Фраза смешивается с сессионным ключом — без неё расшифровка даёт только "цифровой мусор". Даже при взломе устройства или доступе спецслужб к данным, без второй фразы прочесть переписку невозможно.

### ⚖️ Proof-of-Trust
Многомерный механизм консенсуса, учитывающий не только баланс сервера (30%), но и его uptime (20%), отсутствие штрафов (10%), активность (20%), скорость реакции (15%) и стаж (5%). Это не позволяет "богатым" доминировать — любой стабильный и честный сервер имеет вес в сети.

### 🏠 Мотивация NAT-серверов
Впервые в блокчейн-сетях: серверы за NAT (домашние ПК, офисные машины) получают **70% комиссии** за инициированные транзакции — значительно больше, чем валидаторы с белым IP! Это создаёт стимул для массового принятия и делает сеть по-настоящему распределённой.

## Экономическая модель

### Монета DNET
- **Тикер:** DNET
- **Десятичных знаков:** 8 (1 DNET = 100,000,000 сатоши)
- **Эмиссия:** только за создание новых серверов (никаких ICO, премайнинга или "командных кошельков")

### Прогрессивная эмиссия со сжиганием

| Диапазон серверов | Эмиссия за сервер | Сжигается (50%) | В депозит | Реальных монет в обращении (накопленно) |
|-------------------|-------------------|-----------------|-----------|------------------------------------------|
| 1-100 | 1000 DNET | 500 DNET | 500 DNET | 50 000 DNET |
| 101-200 | 800 DNET | 400 DNET | 400 DNET | 90 000 DNET |
| 201-500 | 500 DNET | 250 DNET | 250 DNET | 165 000 DNET |
| 501-1000 | 300 DNET | 150 DNET | 150 DNET | 240 000 DNET |
| 1001-2000 | 150 DNET | 75 DNET | 75 DNET | 315 000 DNET |
| 2001-5000 | 75 DNET | 37.5 DNET | 37.5 DNET | 427 500 DNET |
| 5001-7500 | 40 DNET | 20 DNET | 20 DNET | 477 500 DNET |
| 7501-8000 | 30 DNET | 15 DNET | 15 DNET | 485 000 DNET |
| 8001-10000 | 20 DNET | 10 DNET | 10 DNET | 505 000 DNET |
| 10001+ | 10 DNET | 5 DNET | 5 DNET | - |

**Первые 100 серверов** получают депозит по 500 реальных DNET — это справедливая компенсация за риск и вклад в становление сети.

### Распределение комиссии транзакций (0.02%)
- **70% — инициатору** (сервер, первым получивший транзакцию от клиента)
- **10% — валидатору** (сервер, включивший транзакцию в блок)
- **20% — сжигается** (дефляционный механизм)

Никакого "фонда развития" или кошельков команды — только честное распределение между участниками сети. Такое распределение создаёт реальную мотивацию для массового запуска серверов за NAT, делая сеть по-настоящему децентрализованной.

## Два типа участников

### Серверы (Node / Miner)
Единое приложение, которое выполняет три функции:
- **Майнинг** — участие в консенсусе Proof-of-Trust, создание блоков
- **Сервер знакомств** — помощь клиентам в обнаружении друг друга
- **Обслуживание клиентов** — приём, хранение и пересылка сообщений

**Требования:**
- Минимальные: 2 ядра, 4 ГБ RAM, 100 ГБ диска
- Рекомендуемые: белый IP (для полного функционала)
- Даже за NAT можно участвовать и зарабатывать на инициации транзакций

### Клиенты (Messenger)
Легкое приложение для конечных пользователей:
- Бесплатные текстовые сообщения с E2EE-шифрованием
- Опциональный кошелёк для отправки/получения DNET
- Создание и исполнение смарт-контрактов
- Уникальная функция "двойной ключ" для сверхприватных чатов

**Регистрация:** email + пароль + сид-фраза (12 слов) для восстановления.

## Дорожная карта

*Проект находится в поиске команды разработчиков. Указанные сроки — от момента формирования команды.*

### Этап 1: Прототипирование (1-2 квартала после старта разработки)

| Задача | Технологии | Ожидаемый результат |
|--------|------------|---------------------|
| Сервер знакомств (прототип) | Python + FastAPI | Рабочий rendezvous-сервер |
| Базовый блокчейн | Python + Cryptography | Транзакции, блоки, хеширование |
| Механизм консенсуса (Proof-of-Trust) | Python + asyncio | Тестовая сеть с 10-20 серверами |
| TUI-клиент | Textual | Базовый интерфейс для тестирования |
| Публикация open-source кода | GitHub | Репозиторий с документацией |

**Результат этапа:** Рабочая тестовая сеть (testnet) для ранних пользователей.

### Этап 2: Альфа-версия (1-2 квартала)

| Задача | Технологии | Ожидаемый результат |
|--------|------------|---------------------|
| Переход на libp2p | Go/Rust или py-libp2p | Устойчивая P2P-сеть |
| Полноценный консенсус | - | Proof-of-Trust с многомерной метрикой |
| Смарт-контракты (escrow) | VM-движок | Работающие контракты с невозвратом |
| Клиент с кошельком | - | Интеграция DNET |
| Приватный чат с двойным ключом | - | Уникальная фишка готова |

**Результат этапа:** Закрытая альфа-версия для первых 100 серверов.

### Этап 3: Бета и Mainnet (1-2 квартала)

| Задача | Технологии | Ожидаемый результат |
|--------|------------|---------------------|
| Аудит безопасности | Внешние эксперты | Исправление уязвимостей |
| Запуск mainnet | - | Основная сеть с реальными монетами |
| Первые 100 серверов | - | Приглашённые майнеры |
| Начальное распределение | - | Прогрессивная эмиссия со сжиганием |

**Результат этапа:** Работающая основная сеть с сообществом.

### Этап 4: Развитие экосистемы (после запуска mainnet)

| Задача | Приоритет | Ожидаемый результат |
|--------|-----------|---------------------|
| Графический клиент (Qt) | Высокий | Удобный интерфейс для масс |
| Мобильные приложения | Высокий | iOS/Android клиенты |
| Инструменты для разработчиков | Средний | SDK, API, документация |
| Биржевые листинги | Низкий | Торговля DNET (опционально) |
| Групповые чаты | Средний | Расширение функционала |
| Файлы и медиа | Низкий | IPFS-интеграция |

**Результат этапа:** Полноценная экосистема с тысячами пользователей.
